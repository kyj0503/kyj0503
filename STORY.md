#### 2.2.3. Just Plan It!
[웹사이트](https://justplanit.site/), [Github](https://github.com/kyj0503/jandi_plan_backend)

**기간:** 2025. 02. ~ 2025. 03.

**역할:** 백엔드 팀장 (FE 2, BE 2)

여행 계획 플래너 및 커뮤니티 플랫폼입니다.

* **OAuth2 심층 학습 및 Stateless CSRF 방어 로직 설계**
    * **문제:** Google Cloud Run과 같은 서버리스 환경 특성상 **세션(`HttpSession`)을 유지하기 어려워**, 기존의 세션 기반 인증 방식을 사용할 수 없었습니다.
    * **해결:** 대안으로 **JWT**를 도입했으나, 표준 라이브러리인 `spring-boot-starter-oauth2-client`가 세션을 사용하는 Stateful 방식임을 파악했습니다. 이에 라이브러리를 사용하지 않고 **`RestTemplate`을 활용해 인증 흐름을 직접 구현**했습니다. 특히 CSRF 방지용 `state` 값까지 JWT로 생성하여 클라이언트에 발급하고 검증하는 **완전한 Stateless OAuth2 인증 로직**을 설계했습니다.
    * **성과:** 별도의 세션 저장소(Redis 등) 없이도 안전한 소셜 로그인을 구축하여 인프라 비용을 절감하고, OAuth2 프로토콜의 동작 원리를 깊이 있게 체득했습니다.

* **데이터 정합성 보장 및 고아 리소스 관리**
    * **문제:** 게시글 작성 중 이미지를 업로드했으나 최종 저장을 하지 않고 이탈할 경우 스토리지(GCS)에 '고아 객체'가 남는 문제가 있었습니다. 또한, 게시글 삭제 시 DB 트랜잭션은 롤백되었으나 이미지는 이미 삭제되어 버리는 정합성 문제가 우려되었습니다.
    * **해결:**
        1.  **임시 ID 시스템:** 게시글 작성 시 **음수 ID**를 임시 발급하여 이미지를 매핑하고, 최종 저장 시 실제 ID로 업데이트하는 로직을 구현했습니다.
        2.  **트랜잭션 동기화:** `TransactionSynchronizationManager`를 활용하여 DB 트랜잭션이 완전히 커밋된 직후(`afterCommit`)에만 GCS 이미지 삭제 요청을 보내도록 하여, 롤백 시 데이터 소실을 방지했습니다.
        3.  **자동 정리 스케줄러:** 매 정각마다 `NegativeImageCleanupJob`이 실행되어, 최종 저장되지 않고 남겨진 음수 ID의 이미지들을 GCS와 DB에서 일괄 정리하도록 자동화했습니다.

* **인프라 아키텍처 설계 및 클라우드 마이그레이션 (GCP → AWS)**

    * **문제:** 초기 구축 시 GCP Cloud Run과 Cloud SQL을 도입하여 인프라 관리 소요를 최소화했고, GitHub Actions를 연동하여 코드 푸시 즉시 배포되는 CI/CD 파이프라인을 운영했습니다. 하지만 GCP 프리티어 종료 후 운영 비용이 발생하기 시작했습니다. 
    * **해결:** AWS EC2와 RDS로 전체 인프라를 마이그레이션했습니다. 배포 환경 변화에 맞춰 Jenkins를 도입, Docker 이미지를 빌드하고 GHCR에 푸시한 뒤 EC2에서 실행하는 파이프라인을 재설계했습니다.
    * **성과:** PaaS(서버리스)와 IaaS(인스턴스) 환경을 모두 직접 구축하고 운영하며 각 아키텍처의 트레이드오프를 경험했습니다.
